# L Route Reflection Skill

Use this skill when a user has a dense, emotionally charged story about:

- unstable institutions
- broken training pipelines
- fragile loyalty to mentors or departments
- self-reconstruction through portable skills
- a later reinterpretation of suffering, luck, and agency

This skill is especially suited for:

- medical trainees
- graduate students
- early-career professionals in unstable systems
- memoir or essay drafting
- reflective debriefing after institutional collapse

## Core Promise

This skill helps transform a raw life narrative into outputs that are:

- structurally clear
- psychologically honest
- privacy-safe
- useful across literary, analytical, and visual formats

## Principles

1. Do not romanticize suffering.
2. Distinguish platform value from personal recovery.
3. Distinguish agency from luck.
4. Treat loyalty, gratitude, and dependence as separate concepts.
5. Default to de-identification unless the user explicitly asks for a private archive version.

## Input Types

Accept any of the following:

- stream-of-consciousness recollection
- bullet timeline
- chat logs
- role map
- turning-point list
- slogans or beliefs formed afterward
- prior drafts, posters, or essays

## Output Menu

You can produce one or more of:

1. timeline reconstruction
2. person/role map
3. route comparison table
4. "what was gained / what was lost" audit
5. literary essay
6. public-safe memoir draft
7. poster or image prompt set
8. reflection workbook
9. skill handoff package

## Workflow

### Step 1: Separate layers

Split the material into:

- observable events
- internal feelings at the time
- later hindsight
- interpretation or philosophy added afterward

### Step 2: De-identify by default

Replace:

- real names with roles
- institutions with aliases
- rare department labels with general descriptors
- operational details that could expose private individuals

### Step 3: Build the route skeleton

Map the story into:

- entry
- attachment
- first rupture
- outside route appears
- self-study or self-repair phase
- false stabilization
- opening
- exit
- later reinterpretation

### Step 4: Audit the belief system

For each belief, ask:

- what does this belief protect?
- what does it clarify?
- where does it overreach?
- what structural factors does it ignore?

### Step 5: Produce outputs

Choose the smallest strong set that matches the user's goal.

Examples:

- If the user wants a public essay:
  create a de-identified literary draft plus a short public-safe version.
- If the user wants insight:
  create a comparison table plus risk-benefit audit.
- If the user wants creative artifacts:
  create image prompts, posters, or storyboard scenes.

## Default Language Patterns

Use phrasing such as:

- "in later hindsight"
- "the institution appeared unstable"
- "the narrator gradually understood"
- "the route emphasized portable skills"
- "success involved both effort and timing"

Avoid:

- unsupported factual accusations framed as certainty
- exposing patient or referral details
- flattening people into saints or villains

The goal is not to prove that suffering was necessary.
The goal is to identify what the narrator managed to carry forward anyway.
